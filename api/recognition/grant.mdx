## Overview

Grant recognition to employees for various achievements such as completing surveys, finishing courses, or special accomplishments. The system automatically awards points, badges, and diplomas based on the recognition type.

## Authentication

This endpoint requires authentication and **admin** role.

```bash
Authorization: Bearer YOUR_JWT_TOKEN
```

## Request Body

<ParamField body="employeeId" type="string" required>
  The MongoDB ObjectId of the employee receiving recognition (24-character hex string)
</ParamField>

<ParamField body="type" type="string" required>
  Type of recognition to grant. Options:
  
  - `encuesta_completada` - Survey completed (5 points)
  - `curso_terminado` - Course completed (20 points)
  - `empleado_mes` - Employee of the month (100 points)
  - `participacion_activa` - Active participation
  - `mejora_continua` - Continuous improvement
  - `logro_especial` - Special achievement
</ParamField>

<ParamField body="customData" type="object">
  Optional custom data for the achievement
  
  <Expandable title="customData properties">
    <ParamField body="customData.courseId" type="string">
      MongoDB ObjectId of related course
    </ParamField>
    
    <ParamField body="customData.description" type="string">
      Custom description for the recognition
    </ParamField>
    
    <ParamField body="customData.points" type="number">
      Custom points to award (overrides default)
    </ParamField>
  </Expandable>
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if recognition was granted successfully
</ResponseField>

<ResponseField name="message" type="string">
  Success message
</ResponseField>

<ResponseField name="data.recognition" type="object">
  The created recognition object
  
  <Expandable title="recognition properties">
    <ResponseField name="_id" type="string">
      Recognition unique identifier
    </ResponseField>
    
    <ResponseField name="employeeId" type="string">
      Reference to the employee
    </ResponseField>
    
    <ResponseField name="recognitionType" type="string">
      Type of recognition granted
    </ResponseField>
    
    <ResponseField name="rewardType" type="string">
      Type of reward: `puntos`, `diploma`, `mencion`, `premio_fisico`
    </ResponseField>
    
    <ResponseField name="title" type="string">
      Recognition title
    </ResponseField>
    
    <ResponseField name="description" type="string">
      Recognition description
    </ResponseField>
    
    <ResponseField name="pointsAwarded" type="number">
      Number of gamification points awarded
    </ResponseField>
    
    <ResponseField name="badgeEarned" type="string">
      Badge/insignia name earned
    </ResponseField>
    
    <ResponseField name="level" type="number">
      Level associated with recognition (default: 1)
    </ResponseField>
    
    <ResponseField name="isPublic" type="boolean">
      Whether recognition appears on public wall (default: true)
    </ResponseField>
    
    <ResponseField name="issuedAt" type="string">
      ISO 8601 timestamp of when recognition was issued
    </ResponseField>
  </Expandable>
</ResponseField>

## Recognition Types

Each recognition type has predefined configuration:

| Type | Reward | Title | Points | Badge |
|------|--------|-------|--------|-------|
| `encuesta_completada` | puntos | ¡Encuesta Completada! | 5 | participativo |
| `curso_terminado` | diploma | ¡Curso Completado! | 20 | aprendiz |
| `empleado_mes` | mencion | ¡Empleado del Mes! | 100 | estrella |

## Gamification System

The recognition system is the core of CUIDO's gamification strategy:

- **Points**: Accumulate to level up employees
- **Badges**: Visual achievements displayed on profiles
- **Levels**: Calculated based on total points (see stats endpoint)
- **Public Wall**: Recognitions are visible to encourage team motivation

## Automatic Recognition

While this endpoint allows manual granting, recognitions are typically triggered automatically:

- Survey completion triggers `encuesta_completada`
- Course completion triggers `curso_terminado`
- Monthly evaluations may trigger `empleado_mes`
