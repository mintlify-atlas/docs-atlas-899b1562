## Introduction

The Employees API allows you to manage healthcare workers in your organization. Each employee record includes personal information, job details, wellness metrics for burnout prevention, and gamification features to encourage engagement.

## Employee Schema

The Employee model contains the following main sections:

<Expandable title="Employee Object Structure">
  <ResponseField name="_id" type="string">
    Unique MongoDB identifier for the employee
  </ResponseField>
  
  <ResponseField name="personalInfo" type="object">
    Personal information about the employee
    <Expandable title="personalInfo fields">
      <ResponseField name="name" type="string" required>
        Full name of the employee (max 100 characters)
      </ResponseField>
      <ResponseField name="email" type="string" required>
        Email address (unique, lowercase)
      </ResponseField>
      <ResponseField name="identification" type="string" required>
        National ID or employee identification number (unique)
      </ResponseField>
      <ResponseField name="phoneNumber" type="string">
        Contact phone number
      </ResponseField>
    </Expandable>
  </ResponseField>

  <ResponseField name="jobInfo" type="object">
    Employment and role information
    <Expandable title="jobInfo fields">
      <ResponseField name="position" type="string" required>
        Employee position. One of: `medico`, `enfermero`, `auxiliar_enfermeria`, `administrativo`, `otro`
      </ResponseField>
      <ResponseField name="department" type="string" required>
        Department assignment. One of: `urgencias`, `hospitalizacion`, `consulta_externa`, `administracion`, `otro`
      </ResponseField>
      <ResponseField name="startDate" type="date" required>
        Employment start date (ISO 8601 format)
      </ResponseField>
      <ResponseField name="shift" type="string">
        Work shift. One of: `mañana`, `tarde`, `noche`, `rotativo` (default: `mañana`)
      </ResponseField>
    </Expandable>
  </ResponseField>

  <ResponseField name="hospitalId" type="string" required>
    Identifier of the hospital this employee belongs to
  </ResponseField>

  <ResponseField name="wellnessMetrics" type="object">
    Metrics for monitoring employee wellness and burnout risk
    <Expandable title="wellnessMetrics fields">
      <ResponseField name="currentMoodScore" type="number">
        Current mood rating (default: 3)
      </ResponseField>
      <ResponseField name="averageWorkload" type="number">
        Average workload score (default: 3)
      </ResponseField>
      <ResponseField name="teamSupportScore" type="number">
        Team support perception score (default: 3)
      </ResponseField>
      <ResponseField name="satisfactionScore" type="number">
        Job satisfaction score (default: 5)
      </ResponseField>
      <ResponseField name="riskLevel" type="string">
        Burnout risk level. One of: `bajo`, `medio`, `alto` (default: `bajo`)
      </ResponseField>
    </Expandable>
  </ResponseField>

  <ResponseField name="gamification" type="object">
    Gamification data to encourage platform engagement
    <Expandable title="gamification fields">
      <ResponseField name="totalPoints" type="number">
        Total points earned (default: 0)
      </ResponseField>
      <ResponseField name="currentStreak" type="number">
        Current consecutive days streak (default: 0)
      </ResponseField>
      <ResponseField name="maxStreak" type="number">
        Maximum streak achieved (default: 0)
      </ResponseField>
      <ResponseField name="level" type="number">
        Current level based on points (default: 1)
      </ResponseField>
      <ResponseField name="badges" type="array">
        Array of earned badges
        <Expandable title="badge object">
          <ResponseField name="name" type="string">
            Badge name
          </ResponseField>
          <ResponseField name="earnedAt" type="date">
            Date when badge was earned
          </ResponseField>
          <ResponseField name="description" type="string">
            Badge description
          </ResponseField>
        </Expandable>
      </ResponseField>
    </Expandable>
  </ResponseField>

  <ResponseField name="isActive" type="boolean">
    Whether the employee is currently active (default: true)
  </ResponseField>

  <ResponseField name="createdAt" type="date">
    Timestamp when the employee was created
  </ResponseField>

  <ResponseField name="updatedAt" type="date">
    Timestamp when the employee was last updated
  </ResponseField>
</Expandable>

## Authentication

All employee endpoints require authentication. Include your access token in the Authorization header:

```bash
Authorization: Bearer YOUR_ACCESS_TOKEN
```

## Authorization

Some endpoints require admin privileges:
- **Create Employee**: Admin only
- **Update Employee**: Admin only
- **Get Employee**: All authenticated users
- **List Employees**: All authenticated users

## List All Employees

Retrieve a paginated list of active employees with optional filtering.

### Endpoint

```
GET /api/employees
```

### Query Parameters

<ParamField query="page" type="number">
  Page number for pagination. Default: 1
</ParamField>

<ParamField query="limit" type="number">
  Number of employees per page. Default: 10
</ParamField>

<ParamField query="department" type="string">
  Filter by department. Options: `urgencias`, `hospitalizacion`, `consulta_externa`, `administracion`, `otro`
</ParamField>

<ParamField query="position" type="string">
  Filter by position. Options: `medico`, `enfermero`, `auxiliar_enfermeria`, `administrativo`, `otro`
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Indicates if the request was successful
</ResponseField>

<ResponseField name="data.employees" type="array">
  Array of employee objects with selected fields (personalInfo, jobInfo, currentStatus)
</ResponseField>

<ResponseField name="data.pagination" type="object">
  Pagination metadata
  
  <Expandable title="pagination properties">
    <ResponseField name="page" type="number">
      Current page number
    </ResponseField>
    <ResponseField name="limit" type="number">
      Number of items per page
    </ResponseField>
    <ResponseField name="total" type="number">
      Total number of employees matching the query
    </ResponseField>
    <ResponseField name="pages" type="number">
      Total number of pages
    </ResponseField>
  </Expandable>
</ResponseField>

### Example Request

<CodeGroup>
```bash cURL
curl -X GET 'https://api.example.com/api/employees?page=1&limit=10&department=urgencias' \
  -H 'Authorization: Bearer YOUR_JWT_TOKEN'
```

```javascript JavaScript
const response = await fetch('https://api.example.com/api/employees?page=1&limit=10&department=urgencias', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
});
const data = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://api.example.com/api/employees',
    params={'page': 1, 'limit': 10, 'department': 'urgencias'},
    headers={'Authorization': f'Bearer {token}'}
)
data = response.json()
```
</CodeGroup>
